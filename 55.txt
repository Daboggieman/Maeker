# Maker Studio – HYBRID (n8n + Python) ROADMAP

This roadmap outlines the dual-track implementation using **n8n** for orchestration and **Python** for specialized processing.

## PHASE 1: HYBRID INFRASTRUCTURE
- [x] **1.1 Orchestration Setup (n8n)**
    - [x] Install n8n locally and set up basic Webhook triggers.
    - [x] Create folder structure for data hand-off (e.g., `/jobs/pending`).
- [x] **1.2 Processing Setup (Python)**
    - [x] Initialize Python environment and core utility classes (`FFmpegEngine`, `AIAgent`).
    - [x] Create simple Python script to listen for n8n triggers or a job queue.
- [x] **1.3 Database & Asset Links**
    - [x] Link n8n and Python to a shared MongoDB Atlas instance.
    - [x] Configure shared local storage for media assets.

## PHASE 2: DATA & INTELLIGENCE PIPELINE
- [ ] **2.1 Sourcing (n8n)**
    - [ ] Build n8n workflows for News APIs and Manual Topic injection.
    - [ ] Pass new topics to Python for processing.
- [x] **2.2 AI Creative Engine (Python)**
    - [x] Develop expert prompt templates for News, Bible, and History in Python.
    - [x] **Implement Multi-Agent Fact-Checking**: Build a secondary LLM pass to verify historical/biblical dates and events.
    - [x] **Dynamic Hook Generator**: Generate 3 high-impact "Hook" variations for Shorts/TikTok content.

## PHASE 3: PRODUCTION & OUTPUT
- [x] **3.1 Audio-Visual Generation (Python)**
    - [x] Python script for Voiceovers — **ElevenLabs primary** (`eleven_turbo_v2_5`), Edge TTS fallback.
    - [x] **Interactive Voice Picker**: CLI tool fetches top 50 bold male narrative voices from ElevenLabs, user selects by number.
    - [x] Implement multi-scene video assembly via MoviePy (Long-form & Vertical).
    - [x] **AI Image Generation**: Pollinations API with model failover (`flux` → `zimage` → `imagen-4`).
- [x] **3.2 Interactive CLI Launcher (`run_job.bat`)**
    - [x] Prompts for Topic, Category (numbered menu), and Voice selection on double-click.
    - [x] Production summary + confirm screen; "N" loops back to topic input.
- [ ] **3.3 Control & Approval (n8n)**
    - [ ] n8n dashboard for Approval/Rejection: View/Select best Hook and Approve Fact-Check status.

## PHASE 4: SCALABILITY & POLISH
- [ ] **4.1 Audit & Policy Sync**
    - [ ] Sync job status between n8n and Python via MongoDB.
    - [ ] Final compliance check for YouTube policies.

## PHASE 5: HYPER-REALISM & EXPANSION
- [/] **5.1 Premium Voice Overhaul (ElevenLabs)**
    - [x] ElevenLabs integrated as primary TTS with interactive voice selection.
    - [ ] Implement Voice Cloning for a unique "Studio Identity".
- [ ] **5.2 Cinematic Video Generation**
    - [ ] Build logic to replace static images with AI-generated cinematic clips.

---

### Current Technical Configuration (As of Phase 3 Implementation)
- **Primary LLM**: Groq (`llama-3.3-70b-versatile`)
- **Secondary LLM**: OpenRouter (`google/gemini-2.5-flash`)
- **Voice (Primary)**: ElevenLabs `eleven_turbo_v2_5` — interactive picker, top 50 bold male narrative voices
- **Voice (Fallback)**: Edge TTS (`en-US-ChristopherNeural`)
- **Image Gen**: Pollinations — `flux` → `zimage` → `imagen-4` (failover chain)
- **Video Assembly**: MoviePy (multi-scene, 16:9 / 9:16)
- **Launcher**: `run_job.bat` — fully interactive CLI (topic, category, voice)
